## 8. 当前问题与风险

- 风险 1：首版范围覆盖全资源、全表单和全高危操作，后续实现必须严格按阶段拆分。
- 风险 2：SQLite 适合单实例快速版，不适合多实例和高并发审计增长。
- 风险 3：前端 service / adapter / mock / ViewModel 已通过 `/check`，后端需要严格按该契约实现真实 API。
- 风险 4：后端工程、真实 API、数据库持久化、Kubernetes API 代理和前后端联调尚未开始。
- 风险 5：创建表单目前只覆盖 Deployment、Service、ConfigMap、Secret、PVC；其中 Deployment 已补充多普通容器、Deployment 基本/元数据、Deployment 更新策略、Pod 安全身份、Pod / 容器 securityContext、Pod 存储（ConfigMap/Secret/PVC/emptyDir）、生命周期、Pod 调度策略、Init 容器、模板 YAML、字号控制、关联 ConfigMap/Secret/PVC 叠层创建、常规/超宽工作区和 YAML 本地基础校验；普通容器与 Init 容器均已收敛为容器列表 + 页签配置模式，其余资源仍保留 YAML 创建或基础表单，待用户确认体验后再按同一 schema 机制扩展。
