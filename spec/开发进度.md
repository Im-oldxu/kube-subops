## 8. 当前问题与风险

- 风险 1：首版范围覆盖全资源、全表单和全高危操作，后续实现必须严格按阶段拆分。
- 风险 2：SQLite 适合单实例快速版，不适合多实例和高并发审计增长。
- 风险 3：前端 service / adapter / mock / ViewModel 已通过 `/check`，后端需要严格按该契约实现真实 API。
- 风险 4：后端工程、真实 API、数据库持久化、Kubernetes API 代理和前后端联调尚未开始。
