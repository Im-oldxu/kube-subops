# 需求文档

## 1. 基本信息

- 需求名称：Kubernetes Web 管理平台
- 需求编号：PRD-20260527-001
- 提出人：用户
- 所属项目：kube-subops
- 优先级：P0
- 目标版本：v1.0
- 提出日期：2026-05-27
- 负责人：待定
- 项目类型：全栈应用
- 阶段需求：需要前端、需要后端、需要 UI

## 2. 背景

平台 / SRE 团队需要一个可通过浏览器使用的 Kubernetes 管理控制台，用于统一接入多个 Kubernetes 集群，查看资源状态，执行常用运维操作，并通过平台权限、安全确认和审计降低误操作风险。

平台应支持通过 kubeconfig、Token、ServiceAccount Token 等方式添加 Kubernetes 集群。连接成功后，系统可以读取集群资源信息，并在 Web 页面中展示和管理。

## 3. 目标

本需求希望达到的结果：

- 目标 1：支持多集群接入、切换、连接检测和资源查看。
- 目标 2：覆盖 Kubernetes 常见资源的列表、详情、YAML、创建、编辑、删除和刷新能力。
- 目标 3：支持工作负载、Pod、Node 等高频运维对象的专项管理能力，包括日志、终端、扩缩容、重启、镜像更新、回滚、drain 等。
- 目标 4：提供平台 RBAC、敏感凭据安全存储、高风险操作二次确认、本地持久审计等安全能力。
- 目标 5：以 Docker Compose 方式交付首版 Web 控制台，平台自身数据服务端持久化。

## 4. 非目标

本次明确不包含的内容，避免范围蔓延：

- 不包含 Helm Chart、Kubernetes 集群内安装包或 Agent 回连模式。
- 不包含桌面客户端、移动 App 或 PWA 安装体验。
- 不包含外部 OIDC / LDAP / AD 单点登录。
- 不包含外部 SIEM、Webhook 或第三方审计系统对接。
- 不承诺复杂任务中心、后台重试队列或跨步骤自动化编排。
- 不承诺备份 / 恢复脚本或备份 / 恢复文档作为 v1.0 验收项。

## 5. 用户场景

描述真实使用场景：

- 场景 1：平台管理员添加多个 Kubernetes 集群，配置平台用户、角色和集群访问范围。
- 场景 2：SRE 运维人员在 Web 控制台中切换集群和 Namespace，查看资源列表、详情、事件、YAML 和关联资源。
- 场景 3：运维人员对工作负载执行创建、编辑、删除、扩缩容、重启、镜像更新、滚动状态查看和回滚。
- 场景 4：运维人员查看 Pod 容器状态、事件、实时日志，并在授权后进入容器终端排障。
- 场景 5：运维人员查看 Node 状态、容量、标签、污点、节点 Pod、节点事件，并执行 drain、cordon、uncordon 等高风险操作。
- 场景 6：审计用户查看平台操作记录，追踪谁在何时对哪个集群、Namespace、资源执行了什么操作以及结果。

## 6. 功能需求

### 6.1 集群管理

- 功能描述：支持添加、查看、测试、更新和删除 Kubernetes 集群连接配置。
- 触发方式：用户进入多集群管理页面，通过页面操作添加或维护集群。
- 输入：集群名称、API Server 地址、kubeconfig、Token、ServiceAccount Token、集群描述等。
- 输出：集群列表、连接测试结果、当前集群上下文和错误提示。
- 业务规则：
  - 平台后端使用保存的 kubeconfig、Token 或 ServiceAccount Token 直连目标集群 API Server。
  - Docker Compose 部署后的平台后端网络必须能够访问目标集群 API Server。
  - 集群凭据由平台统一托管，不按个人 Kubernetes 凭据授权。
  - 凭据更新、删除和连接测试必须记录审计。

### 6.2 布局与导航

- 功能描述：提供 Kubernetes 管理控制台的基础页面框架。
- 触发方式：用户登录后进入 Web 控制台。
- 输入：当前登录用户、当前集群、当前 Namespace、权限信息。
- 输出：顶部栏、左侧边栏和主内容区。
- 业务规则：
  - 顶部栏展示当前集群、当前 Namespace、全局搜索入口、用户信息和退出入口。
  - 左侧边栏按照 Kubernetes 资源类型分组，支持一级菜单和二级菜单。
  - 左侧边栏只负责导航，不直接放置创建、编辑、删除、扩缩容、重启、回滚、日志、终端等具体操作按钮。
  - 主内容区根据当前菜单展示资源列表、资源详情、操作表单、YAML 编辑器、日志视图或终端视图。

左侧导航结构为：

- 集群总览
  - 集群状态
  - 资源关系图
- Pod
- 工作负载
  - Deployment
  - StatefulSet
  - DaemonSet
  - ReplicaSet
  - Job
  - CronJob
- 服务与网络
  - Service
  - Ingress
  - Endpoint
  - EndpointSlice
  - NetworkPolicy
- 配置与密钥
  - ConfigMap
  - Secret
- 存储管理
  - StorageClass
  - PersistentVolume
  - PersistentVolumeClaim
- 访问控制
  - ServiceAccount
  - Role
  - ClusterRole
  - RoleBinding
  - ClusterRoleBinding
- 其他资源
  - Node
  - Namespace
  - Event
- 多集群管理
- 系统设置

### 6.3 资源通用列表页

- 功能描述：所有指定资源都必须具备列表页。
- 触发方式：用户点击左侧资源菜单。
- 输入：当前集群、当前 Namespace、资源类型、名称关键词、标签、状态、分页参数。
- 输出：资源列表、筛选结果、分页信息、操作入口和错误提示。
- 业务规则：
  - 列表页支持按集群、Namespace、名称、标签、状态进行筛选和搜索。
  - 列表页支持分页、刷新、查看详情、查看 YAML、创建、编辑和删除。
  - 集群级资源不应强制要求 Namespace 筛选；命名空间级资源应支持 Namespace 筛选。
  - 无权限资源或操作应隐藏或置灰。
  - 高风险操作不作为左侧菜单按钮出现，必须位于资源列表页或详情页。

### 6.4 资源通用详情页

- 功能描述：所有指定资源都必须具备详情页。
- 触发方式：用户从资源列表进入详情。
- 输入：当前集群、Namespace、资源类型、资源名称。
- 输出：基本信息、状态、标签、注解、关联资源、事件、YAML 和操作入口。
- 业务规则：
  - 详情页展示 metadata、spec、status 的关键字段。
  - 详情页展示标签、注解、关联资源、事件和 YAML。
  - 删除、重启、回滚、drain 节点、查看 Secret 明文、进入 Pod 终端、应用 YAML 等高风险操作必须二次确认。
  - 所有关键操作必须记录审计。

### 6.5 工作负载管理

- 功能描述：支持 Deployment、StatefulSet、DaemonSet、ReplicaSet、Job、CronJob 的管理。
- 触发方式：用户进入工作负载分组下的对应资源页面。
- 输入：工作负载资源定义、镜像、replicas、更新策略、调度配置、YAML 等。
- 输出：工作负载列表、详情、滚动状态、事件、YAML 和操作结果。
- 业务规则：
  - 支持创建、查看、编辑、删除、扩缩容、重启、更新镜像、查看滚动更新状态和回滚。
  - Job / CronJob 等不适用的操作应隐藏或置灰。
  - 扩缩容、重启、更新镜像和回滚必须校验权限并记录审计。
  - 回滚属于高风险操作，必须二次确认。

### 6.6 Pod 管理

- 功能描述：支持 Pod 的查看、排障和基础操作。
- 触发方式：用户进入 Pod 页面或从关联资源进入 Pod 详情。
- 输入：当前集群、Namespace、Pod 名称、容器名称、日志参数、终端参数。
- 输出：Pod 列表、详情、容器状态、事件、日志、实时日志、终端和 YAML。
- 业务规则：
  - 支持查看详情、容器状态、事件、日志、实时日志、进入容器终端、删除 Pod 和查看 YAML。
  - 进入容器终端必须具备权限、二次确认并记录审计。
  - 删除 Pod 属于高风险操作，必须二次确认并记录审计。

### 6.7 服务与网络

- 功能描述：支持 Service、Ingress、Endpoint、EndpointSlice、NetworkPolicy 的管理。
- 触发方式：用户进入服务与网络分组下的对应资源页面。
- 输入：资源定义、端口、选择器、路由规则、网络策略、YAML 等。
- 输出：资源列表、详情、YAML、事件和操作结果。
- 业务规则：
  - 支持列表、详情、YAML、创建、编辑、删除。
  - 删除和 YAML 应用属于高风险操作，必须二次确认并记录审计。

### 6.8 配置与密钥

- 功能描述：支持 ConfigMap 和 Secret 的管理。
- 触发方式：用户进入配置与密钥分组下的对应资源页面。
- 输入：键值配置、Secret 数据、类型、YAML 等。
- 输出：资源列表、详情、YAML、明文查看结果和操作结果。
- 业务规则：
  - ConfigMap、Secret 支持列表、详情、YAML、创建、编辑、删除。
  - 查看 Secret 明文必须二次确认并记录审计。
  - Secret 明文不得在前端长期保存。

### 6.9 存储管理

- 功能描述：支持 StorageClass、PersistentVolume、PersistentVolumeClaim 的管理。
- 触发方式：用户进入存储管理分组下的对应资源页面。
- 输入：存储类、PV、PVC 定义和 YAML。
- 输出：资源列表、详情、状态、关联资源、YAML 和操作结果。
- 业务规则：
  - 支持列表、详情、YAML、创建、编辑、删除。
  - StorageClass、PersistentVolume 按集群级资源处理。
  - PersistentVolumeClaim 按命名空间级资源处理。
  - 删除存储资源属于高风险操作，必须二次确认并记录审计。

### 6.10 访问控制资源

- 功能描述：支持 ServiceAccount、Role、ClusterRole、RoleBinding、ClusterRoleBinding 的管理。
- 触发方式：用户进入访问控制分组下的对应资源页面。
- 输入：RBAC 资源定义、权限规则、绑定对象和 YAML。
- 输出：资源列表、详情、YAML 和操作结果。
- 业务规则：
  - 支持列表、详情、YAML、创建、编辑、删除。
  - Role、RoleBinding、ServiceAccount 按命名空间级资源处理。
  - ClusterRole、ClusterRoleBinding 按集群级资源处理。
  - 对访问控制资源的修改属于高风险操作，必须二次确认并记录审计。

### 6.11 其他资源

- 功能描述：支持 Node、Namespace、Event 的管理与查看。
- 触发方式：用户进入其他资源分组下的对应资源页面。
- 输入：资源名称、状态、标签、污点、事件、YAML 等。
- 输出：资源列表、详情、状态、事件、YAML 和操作结果。
- 业务规则：
  - Node 支持查看状态、资源、标签、污点、节点上的 Pod 和节点事件。
  - Node 支持 drain、cordon、uncordon。
  - Namespace 支持列表、详情、YAML、创建、编辑、删除。
  - Event 以查看为主，若 Kubernetes API 不支持或不适合编辑删除，应隐藏或置灰不适用操作。
  - Node drain、cordon、uncordon、Namespace 删除属于高风险操作，必须二次确认并记录审计。

### 6.12 集群总览与资源关系图

- 功能描述：支持集群状态查看和资源关系图。
- 触发方式：用户进入集群总览页面。
- 输入：当前集群、Namespace、资源对象、关联关系和指标数据。
- 输出：集群状态、资源统计、资源关系图和指标降级提示。
- 业务规则：
  - 资源关系图展示集群、Namespace、工作负载、ReplicaSet、Pod、Service、Ingress、PVC 等资源间的关联关系。
  - 指标展示采用可选指标源；目标集群存在 Metrics API / metrics-server 时展示实时 CPU / 内存用量。
  - Metrics API 不可用时，展示容量、requests、limits 和状态，并提示指标源不可用。

### 6.13 全局搜索

- 功能描述：支持在当前集群范围内搜索 Kubernetes 资源。
- 触发方式：用户使用顶部栏全局搜索入口。
- 输入：关键词、当前集群、资源类型、Namespace。
- 输出：搜索结果列表和跳转入口。
- 业务规则：
  - 首版支持在当前集群范围内按资源名称、类型、Namespace 搜索。
  - 跨集群搜索作为后续增强，不进入 v1.0 验收范围。

### 6.14 系统设置

- 功能描述：支持平台用户、角色、权限点、审计保留周期、平台配置和安全配置管理。
- 触发方式：授权用户进入系统设置页面。
- 输入：用户、角色、权限点、审计保留配置、平台配置。
- 输出：配置列表、详情、编辑结果和审计记录。
- 业务规则：
  - 首版采用内置账号体系。
  - 内置管理员、运维、只读用户三类基础角色。
  - 支持对角色配置自定义权限点。
  - 系统设置相关变更必须记录审计。

## 7. 交互/界面要求

- 页面/入口：登录页、主控制台、顶部栏、左侧导航、集群总览、资源列表、资源详情、YAML 编辑器、日志视图、终端视图、多集群管理、系统设置。
- 终端、部署交付目标与发布产物边界：首版交付 Web 管理后台和 Docker Compose 部署；不交付 PWA、桌面客户端、移动 App、Helm Chart、GitHub Release 安装包或集群内 Agent。
- 部署配置与迁移要求：首版要求平台数据服务端持久化并支持容器数据卷保存；`.env.example -> .env`、quick / production 部署样例、初始化脚本、备份脚本、数据库迁移脚本是否进入交付范围由后续 `/arch-plan` 与 `/deploy` 阶段确认；备份 / 恢复脚本或文档不作为 v1.0 验收项。
- 关键交互：
  - 顶部栏支持当前集群、当前 Namespace、全局搜索、用户信息和退出。
  - 左侧边栏只做资源导航，不放具体操作按钮。
  - 创建、编辑、删除、扩缩容、重启、回滚、查看日志、进入终端等操作放在资源列表页或资源详情页中。
  - 资源列表需支持批量感知，但首版批量操作仅在明确安全的资源上开放；高风险批量删除默认不作为首选交互。
  - YAML 编辑器支持查看、编辑和应用 YAML；应用 YAML 属于高风险操作。
  - 日志视图支持普通日志和实时日志。
  - 终端视图仅对授权用户开放，并在进入前二次确认。
- 状态变化：
  - 操作反馈采用同步结果 + 状态刷新。
  - 提交后展示 Kubernetes API 返回结果，刷新资源状态，记录审计。
  - 长过程状态通过详情页、事件和滚动更新状态查看。
- 异常提示：
  - 集群不可达、Kubernetes API 权限不足、资源不存在、YAML 应用失败、指标源不可用时必须给出明确提示。
  - 所有关键写操作失败时应展示失败原因，不产生误导性成功提示。

  - Kubernetes 数据对象：Pod、Deployment、StatefulSet、DaemonSet、ReplicaSet、Job、CronJob、Service、Ingress、Endpoint、EndpointSlice、NetworkPolicy、ConfigMap、Secret、StorageClass、PersistentVolume、PersistentVolumeClaim、ServiceAccount、Role、ClusterRole、RoleBinding、ClusterRoleBinding、Node、Namespace、Event。
- 新增字段：由后续架构与数据库设计阶段定义。
- 修改字段：当前为新项目，无历史字段修改。
- 涉及接口：
  - 用户登录、退出和当前用户信息接口。
  - 集群管理、连接测试和集群切换接口。
  - Namespace 查询与切换接口。
  - Kubernetes 资源列表、详情、YAML、创建、编辑、删除接口。
  - 工作负载专项操作接口。
  - Pod 日志、实时日志和终端接口。
  - Node 操作接口。
  - Secret 明文查看接口。
  - 平台用户、角色、权限点和系统设置接口。
  - 审计日志查询接口。

- 数据持久化：

  - 前端不得长期明文保存 kubeconfig、Token 或 ServiceAccount Token。

  - 首版要求支持容器数据卷持久保存。
  - 集群资源操作应返回成功 / 失败结果、错误原因，并触发审计记录。

## 9. 权限与安全

- 访问权限：
  - 首版采用内置账号体系，支持用户名 / 密码登录、用户信息展示和退出。
  - 平台权限采用平台 RBAC 为主，不按个人 Kubernetes RBAC 授权。
  - 托管集群凭据的 Kubernetes 权限作为平台可执行能力的实际上限。
  - 内置管理员、运维、只读用户三类基础角色，同时支持对角色配置自定义权限点。
- 数据权限：
  - 平台 RBAC 控制用户可访问的集群、Namespace、资源类型和操作。
  - 无权限资源或操作应隐藏或置灰。
  - 前后端都必须进行权限校验，不能只依赖前端隐藏按钮。
- 安全要求：
  - kubeconfig、Token、ServiceAccount Token 等敏感信息不得在前端明文长期保存。
  - 后端持久化敏感凭据时必须加密或采取等效保护。
  - 登录态、接口鉴权、敏感操作确认、错误信息脱敏和权限一致校验属于基础安全要求。
  - 查看 Secret 明文、进入 Pod 终端、应用 YAML、删除资源、回滚、节点 drain 等高风险操作必须二次确认。
- 审计要求：
  - 本地持久记录登录、退出、集群连接、资源创建、编辑、删除、YAML 应用、终端进入、Secret 明文查看、Node 操作、工作负载回滚等关键操作。
  - 审计记录应至少可追踪用户、时间、集群、Namespace、资源类型、资源名称、操作、结果和失败原因。
  - 审计保留周期应可配置。

## 10. 验收标准

- [ ] 能通过 Docker Compose 启动平台 Web 控制台，并完成首次管理员登录。
- [ ] 能添加至少一个 Kubernetes 集群，完成连接测试，并在顶部栏切换当前集群和 Namespace。
- [ ] 左侧导航结构与需求给定分组一致，且左侧边栏不直接承载资源操作按钮。
- [ ] 所有指定资源均具备列表页和详情页，列表支持集群、Namespace、名称、标签、状态筛选 / 搜索、分页、刷新、查看详情、查看 YAML、创建、编辑、删除。
- [ ] 工作负载支持扩缩容、重启、镜像更新、滚动更新状态查看和回滚，并对不适用资源隐藏或置灰不适用操作。
- [ ] Pod 支持容器状态、事件、日志、实时日志、终端、删除和 YAML 查看。
- [ ] Node 支持状态、资源、标签、污点、节点 Pod、节点事件和 drain / cordon / uncordon。
- [ ] Secret 明文查看、Pod 终端、YAML 应用、删除、回滚、节点 drain 等高风险操作均有二次确认和审计记录。
- [ ] 平台 RBAC 能控制用户可见资源和可执行操作；无权限内容隐藏或置灰。
- [ ] kubeconfig、Token 等敏感信息不在前端长期明文保存，后端持久化具备加密或等效保护。
- [ ] Metrics API 不可用时，页面能降级展示非实时资源信息并给出明确提示。
- [ ] 所有关键写操作失败时能展示失败原因，不产生误导性成功提示。

## 11. 风险与依赖

- 依赖项：
  - 目标集群 API Server 必须能被平台后端网络直连。
  - 实时 CPU / 内存依赖目标集群 Metrics API / metrics-server。
  - 平台权限、凭据加密和审计持久化依赖后续架构和数据设计落地。
- 潜在风险：
  - 全资源 + 全表单 + 全高危操作首版范围较大，后续架构和开发计划必须按完整控制台拆分模块与里程碑。
  - 跨网络、代理、跳板和 Agent 模式不属于首版，复杂企业网络下可能无法直接连接目标集群。
  - 无指标源时只能降级展示资源容量、requests、limits 和状态。
  - Kubernetes 版本差异、CRD 扩展资源、资源字段差异和 API 权限失败需要在后续架构阶段明确兼容策略。
  - 仅平台 RBAC 会弱化“按个人 Kubernetes RBAC 授权”的语义；当前口径已明确为平台 RBAC 主导，托管凭据权限为上限。
- 后续交付形态预留：
  - Helm Chart、Agent 模式、外部 SSO、外部审计、跨集群搜索、备份 / 恢复脚本、移动端和桌面端均作为后续可选方向，不进入 v1.0。
- 备注：
  - v1.0 以完整 Kubernetes Web 控制台首版处理，不再按轻量 MVP 处理。
