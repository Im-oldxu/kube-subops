## 2026-05-27 v1.0.0

### 新增功能

- 确立 Kubernetes Web 管理平台 v1.0 需求基线，覆盖多集群管理、资源管理、日志终端、YAML 管理、平台 RBAC、操作审计和基础安全保护。

### 优化改进

- 前端资源创建体验增加首批表单化能力：Deployment、Service、ConfigMap、Secret、PVC 支持表单填写与 YAML 双向同步；其他资源暂保留 YAML 创建，待首批体验确认后扩展。
- 根据浏览器批注优化创建弹窗：表单创建 / YAML 编辑选中态更明显，创建弹窗在窄视口不再横向滚动，Deployment 表单补充资源 requests / limits、健康检查、ServiceAccount 和 nodeSelector，并提示复杂 YAML 字段保留在高级配置中。
- 继续优化 Deployment 创建表单：资源限制拆分为 CPU / 内存切换面板，健康检查拆分为 Readiness / Liveness 切换面板并补充延迟、周期、超时、失败阈值和成功阈值，新增 ConfigMap / Secret / emptyDir 挂载配置分组，YAML 编辑区改为随内容自适应高度。
- 继续优化 Deployment 高级创建表单：所有配置分组支持折叠收起，调度策略移动到挂载配置后并拆分 nodeSelector、节点亲和、Pod 亲和、Pod 反亲和和容忍配置，补充 Startup 探针、PostStart / PreStop 生命周期钩子、Web/API/Worker/DB YAML 模板、YAML 字号调整和 ConfigMap / Secret 选择/创建入口。
- 继续优化 Deployment 创建弹窗：ServiceAccount 归位到 Pod 模板，调度策略 key 输入区加宽，YAML 模板明确为可编辑预填并保留已修改镜像，挂载表单只选择已有 ConfigMap / Secret，创建入口切换到真实 ConfigMap / Secret 表单，折叠按钮统一右对齐，并放大弹窗工作区以支持表单和 YAML 并排编辑。
- 继续优化 Deployment 创建体验：创建弹窗增加常规 / 超宽模式以适配不同屏幕；ConfigMap / Secret 改为在当前 Deployment 表单上叠加创建，取消不丢失主表单，创建成功后自动回填挂载名称；YAML 区增加本地语法解析、Kubernetes 基础字段校验、注释忽略说明和高亮预览。
- 继续优化 Deployment Pod 模板：ServiceAccount 调整到环境变量之后；Pod 模板增加普通容器 / Init 容器切换，Init 容器支持基础信息、command / args、环境变量、资源 requests / limits 和挂载配置，并按 Kubernetes 官方语义不配置普通 Init 容器不支持的探针与生命周期钩子；YAML 校验提示合并到 YAML 编辑区，去除独立黑底高亮预览块。
- 继续优化 Deployment Pod 模板上下文：切到 Init 容器时隐藏普通容器专属的资源限制、健康检查、挂载与配置、生命周期钩子分组，只保留 Init 容器自身资源 / 挂载配置；调度策略改名为 Pod 调度策略并说明其作用于整个 Pod。
- 继续优化 Deployment 普通容器编辑体验：普通容器改为与 Init 容器一致的容器列表 + 配置页签模式，支持多个普通容器分别配置基础、端口、环境变量、资源、健康检查、挂载和生命周期；YAML 右侧仅保留语法有效 / 错误状态提示，移除面向用户无必要的基础字段、准入兜底和注释说明胶囊。
- 继续优化 Deployment 表单分类语义：基本信息只保留名称和 Namespace；标签 / 注解独立为元数据分组；ServiceAccount 与自动挂载 Token 独立为 Pod 安全身份分组；Deployment 更新策略独立配置 RollingUpdate/Recreate、maxSurge、maxUnavailable、minReadySeconds、revisionHistoryLimit、progressDeadlineSeconds 和 paused，默认不写入 YAML；创建弹窗分类默认收起。
- 继续优化 Deployment 创建表单信息架构：按 Deployment、Deployment 更新策略、Pod 模板、Pod 安全、Pod 存储、Pod 调度策略顺序重排；Deployment 内部拆分基本信息 / 元数据信息，基本信息包含名称、Namespace 和副本数，默认标签 key 为 app、注解为空；Pod 安全补充 Pod / 容器 securityContext；Pod 存储补充 PVC 挂载和叠层创建；Pod 调度策略合并为节点调度、Pod 调度和容忍三类。
- 继续优化 Deployment Pod 存储模型：Pod 存储卷只生成 `spec.template.spec.volumes`，普通容器和 Init 容器的挂载页签从 Pod Volume 中选择并分别生成 `volumeMounts`；ConfigMap / Secret / PVC 叠层创建统一为“新建并选中”语义。
- 继续优化 Deployment 创建表单结构：容器 SecurityContext 从 Pod 安全移入 Pod 模板的普通容器“容器安全”页签，并按当前普通容器生成 `securityContext`；Pod 模板折叠态补充说明，创建弹窗默认常规宽度，YAML 区移除字号控件。
- 继续精简 Deployment 创建默认 YAML：默认模板和模板切换只生成最小 Deployment 骨架，切换 Web / API / Worker / DB 模板时同步替换对应镜像；普通容器端口配置并入“基础”页签，“容器安全”页签简化为“安全”。
- 继续优化 Deployment 普通容器编辑入口：“添加普通容器”移动到普通容器名称列表右侧，并改为红色按钮，减少新增入口的查找成本。
- 继续补齐 Deployment Pod 级镜像拉取和网络配置：普通容器端口默认空态，`hostPort` 显式勾选后写入；Pod 安全补充 `imagePullSecrets`；Pod 调度补充 `hostNetwork`、`dnsPolicy` 和 `dnsConfig`。
- 继续优化 Deployment Init 容器编辑入口与安全配置：“添加 Init 容器”移动到 Init 容器名称列表右侧并使用红色按钮；普通容器和 Init 容器的移除当前容器按钮统一改为红色危险色；普通 Init 容器补充 `securityContext` 表单和 YAML 解析/生成能力，并按 Kubernetes 官方语义不增加 lifecycle、readinessProbe、livenessProbe 或 startupProbe。
- 继续重构 Deployment Pod 网络与调度分类：`hostNetwork`、`dnsPolicy`、`dnsConfig` 从 Pod 调度中拆出，作为 Pod 安全下方的独立“Pod 网络”；Pod 亲和 / 反亲和独立为“Pod 调度”；`nodeSelector`、节点亲和和污点容忍独立为“节点调度”，并将节点标签选择、节点亲和、污点容忍的表头、空态和按钮改为中文化表达。
- 修复 Deployment YAML 编辑模式模板切换问题：在 YAML 编辑中选择 Web / API / 后台任务 / 轻量 DB 模板时保持当前 YAML 编辑模式，只更新表单状态并重新生成 YAML，不再跳回表单创建后覆盖模板内容。
- 优化 Deployment 调度规则表单布局：Pod 亲和 / 反亲和、节点标签选择、节点亲和和污点容忍的添加按钮统一使用红色主动作；每条规则的删除按钮下移到第二行右侧，字段区按容器宽度换行，避免在常规弹窗宽度下横向溢出。
- 优化 Deployment YAML 编辑区标题布局：将 YAML 语法状态胶囊移动到 YAML 标题旁边，说明文案独立放到下一行，避免状态提示贴近右侧造成文字截断观感。
- 修复 Deployment 调度规则在宽视口窄容器中仍横向溢出的问题：去除按 `2xl` 视口触发的多列规则布局，固定为容器内两列换行；恢复 YAML 字号 12/13/14/16 调整能力，并同步作用于高亮层和输入层。
- 继续收敛 Deployment YAML 工具栏：将字号控制移动到标题行右侧，左侧增加当前表单模式 / YAML 模式徽标，并移除模板描述灰底框和 YAML 下方重复说明文案。
- 优化 Deployment Pod 安全与添加入口：`imagePullSecrets` 从文本输入改为 Secret 多选区，多选后自动以逗号写回并生成 YAML；支持新建 Secret 并自动选中；DNS 选项、Pod 存储卷添加和新建并选中按钮统一为红色主动作。
- 继续优化 Deployment Pod 安全与挂载入口：`imagePullSecrets` 改为下拉式多选，仅展示镜像仓库认证 Secret（`kubernetes.io/dockerconfigjson` / `kubernetes.io/dockercfg`）并保留新建 Secret 自动选中；移除 Pod 存储卷、普通容器挂载和 Init 容器挂载行内重复“新建并选中”入口，统一使用底部新建入口。
