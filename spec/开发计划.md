# 开发计划

> 用于沉淀任务基线、阶段顺序、里程碑、资源协作和验收口径。
> 本文件不维护架构细节、接口契约、数据库结构或 UI 页面细节；这些内容分别以 `spec/架构设计.md`、`spec/API设计.md`、`spec/数据库设计.md`、`spec/UI原型设计.md` 为准。

## 1. 基本信息

- 计划名称：Kubernetes Web 管理平台 v1.0 开发计划
- 所属项目：kube-subops
- 负责人：待定
- 制定日期：2026-05-27
- 目标版本：v1.0.0
- 项目类型：全栈应用

## 2. 开发目标

- 目标 1：完成 Vue 管理后台、Go API 服务、SQLite 持久化和 Kubernetes API 代理的完整工程基线。
- 目标 2：实现多集群、资源管理、YAML、日志终端、工作负载操作、平台 RBAC、审计和系统设置。
- 目标 3：通过事实源一致性检查，确保 PRD、架构、API、数据库、UI 和代码实现闭环。

## 3. 范围说明

### 包含范围

- Web 管理后台和 Docker Compose 单实例运行目标。
- Go + Gin + client-go 后端服务。
- SQLite + sqlc + goose 平台数据库。
- 内置账号、服务端会话、平台 RBAC、集群凭据 AES-GCM 加密、操作审计。
- PRD 中列出的 Kubernetes 资源列表、详情、YAML、创建、编辑、删除及专项运维操作。

### 不包含范围

- Helm Chart、Agent 回连、外部 SSO、外部审计、桌面客户端、移动 App、PWA。
- 备份恢复脚本、GitHub Release 安装包、生产 PostgreSQL 双库交付。
- UI 模板选择和页面映射细节；这些由 `/ui` 阶段沉淀到 `spec/UI原型设计.md`。

## 4. 技术栈摘要

| 维度 | 技术栈 | 版本 / 约束 | 事实源 |
|------|--------|-------------|--------|
| 前端 | Vue 3 + TypeScript + Vite + Vue Router + Pinia；模板为 Sub Admin（`sub-admin`） | 建议 Node.js 20.x、`pnpm@9.15.9`，以 `spec/UI原型设计.md` 和模板 `packageManager` 为准 | `spec/UI原型设计.md` |
| 后端 | Go + Gin + client-go | 精确版本待 `backend/` 依赖文件确认 | `spec/架构设计.md` |
| 数据库 | SQLite + sqlc + goose | 单实例 Compose 主库，启用 WAL 和外键 | `spec/数据库设计.md` |
| 部署方式 | Docker Compose | 具体交付文件由 `/deploy` 生成 | `spec/架构设计.md` |

## 8. 风险评估

| 风险 | 影响 | 应对措施 | 是否阻塞 |
|------|------|----------|----------|
| 首版范围覆盖全资源、全表单和全高危操作 | 高 | 按 UI、前端、后端和检查阶段拆分里程碑，优先稳定通用资源能力 | 否 |
| SQLite 单实例和审计增长边界 | 中 | 审计保留周期、WAL、后续 PostgreSQL 增强预留 | 否 |
| Kubernetes API 版本和权限差异 | 中 | 后端封装 client-go，错误脱敏返回，检查阶段覆盖失败场景 | 否 |
| 模板已确认但尚未复制到 `frontend/` | 中 | 下一阶段先走 `/git-bootstrap` 建立工程起点，再由 `/dev-frontend` 执行模板复制与最小验证 | 否 |
| 集群网络不可达 | 高 | 添加集群连接测试和明确错误提示，跨网络 Agent 模式延期 | 否 |
